## 截图要求

### 技术要求
- 格式：PNG（推荐）或 JPG
- 分辨率：至少 1440x900
- 文件大小：尽量控制在 500KB 以内

### 内容要求
- 使用真实或模拟的数据（不要使用空白页面）
- 确保界面清晰可读
- 避免包含敏感信息
- 保持界面整洁美观

### 命名规范
- 使用小写字母
- 使用连字符分隔单词
- 使用描述性名称
- 例如：`homepage.png`, `reimbursement-list.png`

## 截图工具推荐

### Windows
- Snipping Tool（系统自带）
- ShareX（免费开源）
- Greenshot（免费开源）

### macOS
- Screenshot（系统自带，Cmd+Shift+4）
- CleanShot X（付费）

### 浏览器扩展
- Awesome Screenshot
- FireShot
- Nimbus Screenshot

## 截图后处理

1. **裁剪**: 去除不必要的边缘
2. **压缩**: 使用 TinyPNG 或类似工具压缩图片
3. **标注**: 如需要，可以添加箭头或文字说明
4. **命名**: 按照上述命名规范保存

## 更新 README

添加截图后，确保 README.md 中的图片链接正确：

```markdown
![系统首页](./doc/screenshots/homepage.png)
```

## 注意事项

- 所有截图应该展示系统的最佳状态
- 确保数据的一致性（例如，金额、日期等）
- 定期更新截图以反映最新的界面变化
- 考虑添加深色模式的截图（如果支持）
